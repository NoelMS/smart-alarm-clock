HOW TO TRANSFER
================

Step 1: USB Transfer
  - Plug USB flash drive into Linux
  - Copy: /home/noelm/smart-alarm-clock-sync/ → USB
  - Plug USB into Windows
  - Copy contents to: C:\Users\noelm\opencode\smart-alarm-clock\

Step 2: Verify Files on Windows
  - Open C:\Users\noelm\opencode\smart-alarm-clock\
  - Should see 30 files + data/ + docs/ folders
  - Check smart_alarm_clock.ino exists

Step 3: Open in Arduino IDE
  - File → Open
  - Navigate to C:\Users\noelm\opencode\smart-alarm-clock\smart_alarm_clock.ino
  - Click Open

Step 4: Install Dependencies (if not already done)
  - Board: ESP32 by Espressif Systems
  - Libraries: RTClib, Adafruit_SSD1306, Adafruit_GFX, Adafruit_NeoPixel, 
              AsyncTCP, ESPAsyncWebServer

Step 5: Verify Compilation
  - Sketch → Verify (Ctrl+R)
  - Wait for "Compilation complete"

Step 6: Upload to ESP32
  - Connect ESP32 with external power (5V adapter)
  - Tools → Board → ESP32 Dev Module
  - Tools → Port → Select COM port
  - Sketch → Upload (Ctrl+U)

WHAT EACH SECTION DOES
======================

FIRMWARE CODE:
  - smart_alarm_clock.ino: Main sketch, entry point
  - config.h: All GPIO pins and settings defined here
  - rtc_manager: Time keeping (DS3231 module)
  - display_manager: OLED screen control
  - button_handler: Physical button inputs
  - alarm_manager: Alarm logic (3 independent alarms)
  - alert_controller: Vibration motor + LED control
  - wifi_manager: WiFi connectivity
  - web_server: REST API endpoints

WEB INTERFACE:
  - index.html: Dashboard layout
  - style.css: Responsive styling
  - script.js: Button interactions, API calls

DOCUMENTATION:
  - SETUP_GUIDE.md: Installation steps for Arduino IDE
  - HARDWARE_SETUP.md: Wiring diagrams and connections
  - USER_MANUAL.md: How to use the alarm clock
  - TROUBLESHOOTING.md: Common issues and solutions
  - WINDOWS_ESP32_FIX.md: Windows USB driver fixes

IMPORTANT NOTES
===============

1. ALL files must stay in same folder (including .h and .cpp)
2. data/ folder must be in project directory for SPIFFS upload
3. docs/ folder is reference only (doesn't affect compilation)
4. ESP32 needs EXTERNAL power (5V adapter) to upload
5. Internal USB power is NOT sufficient (causes failures)
